# Micro-Latent RAMA Model Setup

This document defines the **micro-latent part** of the Latent-RAMA model.

The micro-latent model is responsible for generating the high-frequency residual latent \(z_H\) conditioned on the macro latent \(z_L\).

The full latent decomposition is:

\[
z = U(z_L) + z_H
\]

where:

- \(z\): full SD-VAE latent
- \(z_L\): low-frequency macro latent
- \(z_H\): high-frequency residual latent
- \(U\): deterministic upsampling operator

The macro latent \(z_L\) is generated by the flow matching model.  
The micro-latent model learns:

\[
p(z_H \mid z_L)
\]

---

## 1. Position in the Full Pipeline

The full Latent-RAMA generation pipeline is:

```text
Gaussian noise
      ↓
macro flow matching model
      ↓
sampled macro latent z_L
      ↓
context encoder
      ↓
local context vectors c_p
      ↓
micro RAMA model
      ↓
sampled high-frequency residual z_H
      ↓
z_final = U(z_L) + z_H
      ↓
SD-VAE decoder
      ↓
generated image
```

The micro-latent part starts after \(z_L\) is available.

---

## 2. Latent Decomposition

For CelebA 256 with SD-VAE:

```text
image x:         [3, 256, 256]
SD-VAE latent z: [4, 32, 32]
```

Decompose:

\[
z_L = D_{\text{down}}(z)
\]

\[
z_H = z - U(z_L)
\]

For the first setup:

```text
z:       [B, 4, 32, 32]
z_L:     [B, 4, 16, 16]
U(z_L):  [B, 4, 32, 32]
z_H:     [B, 4, 32, 32]
```

Code:

```python
import torch.nn.functional as F

def decompose_latent(z):
    z_l = F.avg_pool2d(z, kernel_size=2, stride=2)
    z_l_up = F.interpolate(
        z_l,
        size=z.shape[-2:],
        mode="bilinear",
        align_corners=False,
    )
    z_h = z - z_l_up
    return z_l, z_h
```

## 3. Micro-Latent Objective

The micro model learns:

\[
p(z_H \mid z_L)
\]

Directly modeling the full joint distribution of \(z_H\) is expensive:

\[
z_H \in \mathbb{R}^{4 \times 32 \times 32}
\]

which contains:

\[
4 \times 32 \times 32 = 4096
\]

scalar dimensions.

The RAMA approximation decomposes this into local patch-wise scalar distributions.

After patchification and random projection:

\[
y_{H,p} = A_p z_{H,p}
\]

the model approximates:

\[
p(z_H \mid z_L)
\approx
\prod_{p=1}^{P}
\prod_{i=1}^{d}
p(y_{H,p,i} \mid c_p, e_i)
\]

where:

- \(p\): patch index
- \(i\): coordinate index inside the patch
- \(z_{H,p}\): residual patch vector
- \(A_p\): frozen random orthogonal basis for patch \(p\)
- \(y_{H,p}\): RAMA-projected residual vector
- \(c_p\): context vector from the context encoder
- \(e_i\): learned coordinate embedding

## 4. Localized RAMA Patchification

For the first setup:

```yaml
latent_shape: [4, 32, 32]
patch_size: 2
patch_grid: [16, 16]
num_patches: 256
patch_dim: 16
```

Each residual patch has shape:

```text
[4, 2, 2]
```

Flattened dimension:

\[
d = 4 \times 2 \times 2 = 16
\]

Patchification transforms:

```text
z_H:     [B, 4, 32, 32]
patches: [B, 256, 16]
```

Code:

```python
def patchify(z_h, patch_size=2):
    # z_h: [B, C, H, W]
    # returns patches: [B, P, d]
    B, C, H, W = z_h.shape
    p = patch_size

    assert H % p == 0
    assert W % p == 0

    patches = z_h.unfold(2, p, p).unfold(3, p, p)
    # [B, C, H/p, W/p, p, p]

    patches = patches.permute(0, 2, 3, 1, 4, 5)
    # [B, H/p, W/p, C, p, p]

    patches = patches.reshape(B, -1, C * p * p)
    # [B, P, d]

    return patches
```

Inverse:

```python
def unpatchify(patches, C, H, W, patch_size=2):
    # patches: [B, P, d]
    # returns z_h: [B, C, H, W]
    B, P, d = patches.shape
    p = patch_size

    h_grid = H // p
    w_grid = W // p

    assert P == h_grid * w_grid
    assert d == C * p * p

    x = patches.reshape(B, h_grid, w_grid, C, p, p)
    x = x.permute(0, 3, 1, 4, 2, 5)
    x = x.reshape(B, C, H, W)

    return x
```

Sanity check:

```python
z_h = torch.randn(8, 4, 32, 32)

patches = patchify(z_h, patch_size=2)
z_h_rec = unpatchify(patches, C=4, H=32, W=32, patch_size=2)

error = (z_h - z_h_rec).abs().max()
print(error)
```

Expected:

```text
0 or close to 0
```

## 5. Cached Spatial Bases

Each patch position \(p\) has a fixed orthogonal matrix:

\[
A_p \in \mathbb{R}^{d \times d}
\]

For the first setup:

```text
P = 256
d = 16
bases: [256, 16, 16]
```

Generate once and reuse during training and inference.

```python
import torch

def make_orthogonal_bases(num_patches, patch_dim, seed=1234, device="cpu"):
    bases = []

    for p in range(num_patches):
        gen = torch.Generator(device=device)
        gen.manual_seed(seed + p)

        M = torch.randn(
            patch_dim,
            patch_dim,
            generator=gen,
            device=device,
        )

        Q, R = torch.linalg.qr(M)

        signs = torch.sign(torch.diag(R))
        signs[signs == 0] = 1
        Q = Q * signs.unsqueeze(0)

        bases.append(Q)

    return torch.stack(bases, dim=0)
```

Save:

```python
bases = make_orthogonal_bases(
    num_patches=256,
    patch_dim=16,
    seed=1234,
    device="cpu",
)

torch.save(bases, "cache/rama_bases_p256_d16.pt")
```

Load:

```python
bases = torch.load("cache/rama_bases_p256_d16.pt")
bases = bases.to(device)
bases.requires_grad_(False)
```

## 6. RAMA Projection and Inverse

Projection:

\[
y_{H,p} = A_p z_{H,p}
\]

Code:

```python
def rama_project(patches, bases):
    # patches: [B, P, d]
    # bases:   [P, d, d]
    # returns y: [B, P, d]
    return torch.einsum("bpd,pde->bpe", patches, bases)
```

Inverse:

\[
z_{H,p} = A_p^T y_{H,p}
\]

Code:

```python
def rama_inverse(y, bases):
    # y:     [B, P, d]
    # bases: [P, d, d]
    # returns patches: [B, P, d]
    return torch.einsum("bpd,ped->bpe", y, bases)
```

Projection sanity check:

```python
patches = torch.randn(8, 256, 16)
bases = make_orthogonal_bases(256, 16)

y = rama_project(patches, bases)
patches_rec = rama_inverse(y, bases)

error = (patches - patches_rec).abs().max()
print(error)
```

Expected:

```text
~1e-6 with float32
```

## 7. Discretization

The discrete micro model quantizes each scalar RAMA coordinate:

\[
y_{H,p,i}
\]

into a token:

\[
v_{p,i} = Q(y_{H,p,i})
\]

where:

\[
v_{p,i} \in \{0,1,\dots,V-1\}
\]

Use clipping range:

\[
[-B, B]
\]

and number of bins:

\[
V
\]

Recommended first setup:

```yaml
num_bins: 256
quant_bound: estimate_from_data
```

Do not blindly use \(B=3.0\). Estimate it from the projected residual values:

\[
B = \text{percentile}(|y|, 99.5\%)
\]

or:

\[
B = \text{percentile}(|y|, 99.9\%)
\]

Quantization:

```python
def quantize(y, B=3.0, V=256):
    # y: [B, P, d]
    # returns integer tokens in [0, V-1]
    y = y.clamp(-B, B)

    y_norm = (y + B) / (2 * B)
    tokens = torch.floor(y_norm * V).long()
    tokens = tokens.clamp(0, V - 1)

    return tokens
```

Dequantization:

```python
def dequantize(tokens, B=3.0, V=256):
    # tokens: [B, P, d]
    # returns bin centers
    return -B + (2 * B / V) * (tokens.float() + 0.5)
```

Quantization-only sanity check:

```python
z_l, z_h = decompose_latent(z)

patches = patchify(z_h, patch_size=2)
y = rama_project(patches, bases)

tokens = quantize(y, B=quant_bound, V=num_bins)
y_hat = dequantize(tokens, B=quant_bound, V=num_bins)

patches_hat = rama_inverse(y_hat, bases)
z_h_hat = unpatchify(patches_hat, C=4, H=32, W=32, patch_size=2)

z_l_up = F.interpolate(z_l, size=(32, 32), mode="bilinear", align_corners=False)
z_hat = z_l_up + z_h_hat
```

Decode \(z_hat\) with the SD-VAE decoder and compare it with the original VAE reconstruction.

If this looks bad, increase `num_bins`, adjust `quant_bound`, or use a continuous density model instead of discretization.

## 8. Context Encoder Input

The micro model conditions on context from \(z_L\):

\[
C = E_\theta(z_L)
\]

Expected shape:

```text
z_L:     [B, 4, 16, 16]
context: [B, 256, D_c]
```

For the first setup:

```yaml
context_dim: 256
num_patches: 256
```

Each patch \(p\) gets one context vector:

\[
c_p \in \mathbb{R}^{D_c}
\]

The context encoder and micro model are trained jointly.

---

## 9. Micro RAMA-Net

The micro RAMA-Net predicts a categorical distribution for each projected coordinate:

\[
P(v_{p,i} \mid c_p, e_i)
\]

where:

- \(c_p\): local context vector
- \(e_i\): learned coordinate embedding
- \(v_{p,i}\): quantized token
- \(i \in \{1,\dots,d\}\)

Input to the MLP:

\[
[c_p, e_i]
\]

Output:

\[
\ell_{p,i} \in \mathbb{R}^{V}
\]

where \(\ell_{p,i}\) are logits over bins.

---

## 10. Micro RAMA-Net Implementation

```python
import torch
import torch.nn as nn


class MicroRAMANet(nn.Module):
    def __init__(
        self,
        context_dim=256,
        patch_dim=16,
        dim_emb_dim=64,
        hidden_dim=512,
        num_bins=256,
        num_layers=4,
    ):
        super().__init__()

        self.patch_dim = patch_dim
        self.num_bins = num_bins

        self.dim_embed = nn.Embedding(patch_dim, dim_emb_dim)

        layers = []
        in_dim = context_dim + dim_emb_dim

        for layer_idx in range(num_layers):
            layer_in = in_dim if layer_idx == 0 else hidden_dim
            layers.append(nn.Linear(layer_in, hidden_dim))
            layers.append(nn.LayerNorm(hidden_dim))
            layers.append(nn.GELU())

        layers.append(nn.Linear(hidden_dim, num_bins))

        self.net = nn.Sequential(*layers)

    def forward(self, context):
        # context: [B, P, D_c]
        # returns logits: [B, P, d, V]
        B, P, D_c = context.shape
        d = self.patch_dim

        dim_ids = torch.arange(d, device=context.device)
        dim_emb = self.dim_embed(dim_ids)
        # [d, dim_emb_dim]

        context = context[:, :, None, :].expand(B, P, d, D_c)
        dim_emb = dim_emb[None, None, :, :].expand(B, P, d, -1)

        x = torch.cat([context, dim_emb], dim=-1)
        # [B, P, d, D_c + dim_emb_dim]

        logits = self.net(x)
        # [B, P, d, V]

        return logits
```

Recommended first config:

```yaml

  context_dim: 256
  patch_dim: 16
  dim_emb_dim: 64
  hidden_dim: 512
  num_layers: 4
  num_bins: 256
```

## 11. Micro-Latent Training Loss

The training target is the quantized RAMA token:

\[
v_{p,i} = Q(y_{H,p,i})
\]

The model predicts logits:

\[
\ell_{p,i} = M_\phi(c_p, e_i)
\]

Training loss:

\[
\mathcal{L}_{micro}
=
\frac{1}{B P d}
\sum_{b=1}^{B}
\sum_{p=1}^{P}
\sum_{i=1}^{d}
\mathrm{CE}
\left(
\ell_{b,p,i},
v_{b,p,i}
\right)
\]

Code:

```python
import torch.nn.functional as F

def micro_ce_loss(logits, tokens, num_bins):
    # logits: [B, P, d, V]
    # tokens: [B, P, d]
    return F.cross_entropy(
        logits.reshape(-1, num_bins),
        tokens.reshape(-1),
    )
```

## 12. Full Micro Training Step

```python
def micro_training_step(
    z,
    context_encoder,
    micro_model,
    bases,
    optimizer,
    patch_size=2,
    quant_bound=3.0,
    num_bins=256,
    context_noise_sigma=0.03,
):
    # z: [B, 4, 32, 32], cached SD-VAE latents

    context_encoder.train()
    micro_model.train()

    z_l, z_h = decompose_latent(z)

    z_l = z_l.detach()
    z_h = z_h.detach()

    if context_noise_sigma > 0:
        z_l_input = z_l + context_noise_sigma * torch.randn_like(z_l)
    else:
        z_l_input = z_l

    patches = patchify(z_h, patch_size=patch_size)
    # [B, 256, 16]

    y = rama_project(patches, bases)
    # [B, 256, 16]

    tokens = quantize(y, B=quant_bound, V=num_bins)
    # [B, 256, 16]

    context = context_encoder(z_l_input)
    # [B, 256, D_c]

    logits = micro_model(context)
    # [B, 256, 16, V]

    loss = micro_ce_loss(logits, tokens, num_bins=num_bins)

    optimizer.zero_grad(set_to_none=True)
    loss.backward()

    torch.nn.utils.clip_grad_norm_(
        list(context_encoder.parameters()) + list(micro_model.parameters()),
        max_norm=1.0,
    )

    optimizer.step()

    return loss.item()
```

## 14. Micro-Latent Sampling

At generation time, the macro flow model provides:

\[
\hat{z}_L
\]

The micro model samples:

\[
\hat{z}_H \sim p_\phi(z_H \mid \hat{z}_L)
\]

Steps:

```text
1. Get context: C = E_theta(z_L)
2. Predict logits: logits = M_phi(C)
3. Sample discrete tokens from categorical distributions
4. Dequantize tokens into y_hat
5. Invert RAMA projection
6. Unpatchify into z_H_hat
```

Code:

```python
@torch.no_grad()
def sample_micro_latent(
    z_l,
    context_encoder,
    micro_model,
    bases,
    latent_channels=4,
    latent_height=32,
    latent_width=32,
    patch_size=2,
    quant_bound=3.0,
    num_bins=256,
    temperature=1.0,
    use_argmax=False,
):
    # z_l: [B, 4, 16, 16]
    # returns z_h_hat: [B, 4, 32, 32]

    context_encoder.eval()
    micro_model.eval()

    context = context_encoder(z_l)
    logits = micro_model(context)
    # [B, P, d, V]

    if temperature != 1.0:
        logits = logits / temperature

    B, P, d, V = logits.shape

    if use_argmax:
        tokens = logits.argmax(dim=-1)
        # [B, P, d]
    else:
        probs = torch.softmax(logits, dim=-1)
        tokens = torch.multinomial(
            probs.reshape(-1, V),
            num_samples=1,
        ).reshape(B, P, d)

    y_hat = dequantize(tokens, B=quant_bound, V=num_bins)
    # [B, P, d]

    patches_hat = rama_inverse(y_hat, bases)
    # [B, P, d]

    z_h_hat = unpatchify(
        patches_hat,
        C=latent_channels,
        H=latent_height,
        W=latent_width,
        patch_size=patch_size,
    )

    return z_h_hat
```

## 17. Sampling Choices

### 17.1 Argmax

```python
tokens = logits.argmax(dim=-1)
```

Pros:

```text
deterministic
less noisy
useful for debugging
```

Cons:

```text
may reduce diversity
may create overly average residuals
```

### 17.2 Categorical Sampling

```text
tokens ~ Categorical(softmax(logits))
```

Pros:

```text
stochastic
more diverse
matches probabilistic generation
```

Cons:

```text
may introduce noise if model is weak
```

### 17.3 Temperature Sampling

\[
\text{logits} = \frac{\text{logits}}{\tau}
\]

Recommended to test:

```yaml
temperature:
  - 0.7
  - 1.0
  - 1.2
```

Lower temperature gives sharper but less diverse outputs.

## 20. Failure Modes

### Failure 1: Quantization-only reconstruction is bad

Likely causes:

```text
too few bins
bad quantization bound
too much clipping
wrong dequantization formula
wrong inverse projection
```

Fix:

```text
increase V from 256 to 512
use percentile(|y|, 99.9%) for B
verify project -> inverse project
```

---

### Failure 2: Micro model cannot overfit tiny batch

Likely causes:

```text
context-patch misalignment
wrong token shape
wrong CE reshape
patchify/unpatchify bug
basis mismatch
learning rate too low
model too small
```

Fix:

```text
test on 16 images
use argmax reconstruction
print all tensor shapes
disable context noise
```

---

### Failure 3: Micro reconstruction is worse than macro-only

Likely causes:

```text
sampled tokens too noisy
model undertrained
temperature too high
quantization too coarse
context encoder too weak
```

Fix:

```text
use argmax first
lower temperature to 0.7
train longer
increase hidden_dim
increase num_bins
```

---

### Failure 4: Works with real z_L but fails with generated z_L

Likely cause:

```text
distribution shift between real z_L and flow-generated z_L
```

Fix:

```text
train macro flow better
add context noise during micro training
condition micro model on slightly noised z_L
use EMA macro model
```

## 22. Summary

The micro-latent part models:

\[
p(z_H \mid z_L)
\]

by transforming local residual patches:

\[
z_{H,p}
\]

into RAMA coordinates:

\[
y_{H,p} = A_p z_{H,p}
\]

then modeling each coordinate independently:

\[
p(y_{H,p,i} \mid c_p, e_i)
\]

In the discrete version, this becomes:

\[
P(v_{p,i} \mid c_p, e_i)
\]
